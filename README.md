# SIFF Picker


- 在线 Demo：https://siff2026-picker.streamlit.app

SIFF Picker 帮助用户在电影节有限的时间和精力约束下，从 SIFF 海量片单中找到可能感兴趣的电影，并生成可进一步了解和购票的推荐片单。

它不是路线规划工具，不解决转场冲突，也不生成最终观影计划。

它帮助用户缩小选择范围，但不替用户决定看什么。

---

## 产品定位

SIFF 每年有上千场放映。对很多观众来说，真正困难的不是排路线，而是：

在有限时间内，如何从海量片单中发现自己可能真正想看的电影。

SIFF Picker 的定位是：

- 帮助用户建立推荐片单
- 帮助用户理解为什么某些电影值得关注
- 保留用户自己的最终判断

产品不替用户决定看什么。

产品的目标是缩小选择范围，而不是给出唯一答案。

---

## MVP 目标

第一版 MVP 优先验证选片逻辑是否有效，而不是验证用户规模。

MVP 需要支持：

- 导入 SIFF CSV
- 获取当前北京时间并过滤已开始场次
- 用户填写可用时间、国家/地区偏好、内容偏好、排除项和补充偏好
- 按硬过滤规则缩小片单
- 按规则打分并生成推荐理由
- 在正分结果不足时使用兜底逻辑补足推荐片单
- 输出电影级推荐片单

---

## 非目标

第一版不做：

- 路线规划
- 转场冲突分析
- 最终观影计划生成
- 自动购票
- 豆瓣同步
- AI 推荐
- embedding / RAG
- 观影人格分析
- 影片年代偏好

## 用户输入

### 一、可用时间，多选

1. 工作日白天
2. 工作日晚间
3. 周末白天
4. 周末晚间
5. 全天不限

可用时间用于硬过滤用户实际无法观看的场次。

### 二、国家/地区偏好，多选

1. 华语
2. 美国
3. 法国
4. 英国
5. 其他欧洲
6. 不限

国家/地区偏好用于加分，不是硬过滤。

### 三、内容偏好，多选

1. SIFF官方精选 / 竞赛片
2. 名导作品
3. 新锐作品
4. 大银幕视听
5. 非虚构
6. 动画 / 短片 / 特别放映
7. 世界电影 / 多元探索
8. 不限

内容偏好用于加分，不是硬过滤。一个 `group` 可以命中多个片单方向。

### 四、排除项，多选

1. 不看恐怖 / 惊悚片
2. 不看超长片（≥3小时）
3. 不看午夜场（24:00后）
4. 不看票价 ≥120
5. 不看无中文字幕场次
6. 不看短片 / 短片集
7. 不限

排除项是硬过滤，不参与得分。

### 五、补充偏好，可跳过

1. 喜欢的导演 / 创作者
2. 印象深刻的作品

补充偏好用于强加分。

## 已删除的输入项

### 影院设施

影院设施题已删除，不再作为输入项，也不再作为独立加分项。

与影院视听相关的判断并入“内容偏好：大银幕视听”。

### 影片年代偏好

第一版暂不做新片 / 老片 / 无所谓。

原因：当前 CSV 中没有稳定年份字段，因此 MVP 第一版暂不做影片年代偏好。

## 推荐对象

推荐对象不是场次，而是电影。系统内部按电影生成候选结果，前端统一展示为“推荐片单”。

也就是说：

- 一部电影只展示一次
- 每部电影关联所有符合筛选条件的可观看场次
- 同一部电影不能因为多个场次而重复出现在主列表中

场次仍然是过滤对象。时间筛选、已开始场次过滤、午夜场排除、票价排除等规则都作用在场次级别。

完成场次级过滤后，系统再按电影聚合。

聚合 key 优先级：

1. 如果 CSV / Screening 模型里有 `officialFilmId`，优先使用 `officialFilmId`
2. 如果模型里有 `film_id`，使用 `film_id`
3. 如果没有 ID，使用 `nameCnBase` / `display_name` / title
4. 如果担心同名电影，使用 title + director 作为兜底 key

---

## 默认前置时间过滤

系统每次运行时，实时获取当前北京时间，也就是 `Asia/Shanghai` 时区下的当前时间。

所有放映开始时间早于或等于当前北京时间的场次，都不进入推荐片单。

规则：

- 当前时间不能写死
- 当前时间应实时获取
- CSV 中的放映时间如果没有时区信息，按 `Asia/Shanghai` 理解
- 只保留放映开始时间 > 当前北京时间的场次

---

## 筛选与得分逻辑

### 硬过滤

1. 默认前置时间过滤
2. 可用时间
3. 排除项

### 加分项

1. 喜欢导演 / 创作者
2. 印象深刻作品
3. 内容偏好
4. 国家/地区偏好

排除项不参与得分，只负责过滤。

---

## 推荐得分规则

推荐得分按电影维度计算，不按场次重复计算。

### 导演 / 创作者

- 完整命中：+6
- 片段命中：+5
- 其他字段命中：+4

### 作品

- 完整命中：+6
- 片段命中：+5
- 其他字段命中：+4

### 片单方向

- 每命中一个用户选择方向：+3
- 单部电影最多计入 2 个方向，上限 +6

### 国家/地区

- 命中：+2
- 单部电影封顶一次

可观看场次数只用于展示和同分排序，不直接加分，避免场次多的热门片天然霸榜。

---

## 模糊匹配规则

喜欢导演 / 创作者、印象深刻作品需要支持基础模糊匹配。

匹配前应做文本标准化，例如：

- 去掉中文间隔号“·”
- 去掉空格
- 去掉英文句点
- 去掉连字符
- 去掉斜杠
- 英文统一小写

例如：

- 用户输入“比利怀尔德”应能命中“比利·怀尔德”
- 用户输入“怀尔德”也应能命中“比利·怀尔德”
- 用户输入“哈利波特”应能命中“哈利·波特与魔法石”

---

## 排除规则

### 恐怖 / 惊悚

当 `group`、`remarks`、`nameCn`、`nameCnBase`、`nameEn` 等文本字段命中“恐怖”或“惊悚”时排除。

### 超长片

当 `lengthMin >= 180` 时排除。

### 午夜场

当散场时间超过 24:00，即 end_time 日期晚于 start_time 日期时排除。

不要用“开始时间 >= 22:00”判断午夜场。

### 票价 ≥120

当 `priceYuan >= 120` 时排除对应场次。

### 无中文字幕

当 `remarks` 包含“无中文字幕”时排除对应场次。

### 短片 / 短片集

当 `group`、`nameCn`、`nameCnBase` 等字段命中“短片”时排除。

---

## 兜底逻辑

触发条件：

- 无任何正分电影
- 或正分电影 < 5

可用时间和排除项属于硬筛选条件，只负责缩小可观看范围，不参与推荐分计算。

国家 / 地区、内容偏好、导演 / 创作者、作品偏好属于会产生推荐分的偏好条件。因此，如果用户只选择了可用时间或排除项，也可能因为没有正分电影而触发兜底逻辑。

兜底逻辑是内部补足机制，不作为前端单独分区；通过兜底补充的电影会与正向推荐结果合并进入最终的推荐片单。

兜底排序：

1. SIFF官方精选 / 竞赛片
2. 名导作品
3. 新锐作品
4. 世界电影 / 多元探索
5. 非虚构
6. 大银幕视听
7. 动画 / 短片 / 特别放映

兜底规则：

- 不显示“推荐分数=0”
- 使用“候选依据”替代“推荐理由”
- 候选依据用于解释为什么这部电影在没有明确偏好命中时仍然进入推荐片单
- 候选依据可以包含默认优先级较高的内容偏好方向和最早可观看场次

---

## 推荐片单展示规则

- 结果区统一显示为“推荐片单”
- 正向推荐结果充足时，前端不额外设置固定 20 部上限，按后端排序结果分页展示
- 触发兜底逻辑时，最终推荐片单最多展示 20 部
- 默认展示前 10 部电影
- 点击“查看更多”后，每次继续展示 10 部
- 兜底模式也按前端分页展示，不一次性铺开 20 部

---

## 排序逻辑

推荐结果按电影维度排序，而不是场次维度排序。

### 正向推荐排序

1. 推荐分数高的在前
2. 同分时，命中导演 / 创作者 / 作品偏好的在前
3. 再同分时，命中内容偏好方向更多的在前
4. 再同分时，最近一场更早的在前
5. 再同分时，可观看场次更多的在前

### 兜底推荐排序

1. 按兜底分类优先级排序
2. 同一分类内，按最早可观看场次时间升序排序

### 放映时间排序

在同一分类 / 同一分数内，按“最早可观看场次时间”升序排序。

但放映时间不能跨类别覆盖排序优先级。

---

## 输出要求

每部电影展示：

- 片名
- 导演
- 国家/地区
- 时长
- 官方单元（如果有）
- 推荐理由或候选依据
- 最早 3 个可观看场次

如果该电影有更多可观看场次，可以展开查看。场次按开始时间升序展示。

场次规格仅作为展示信息，不参与推荐加分。

所有结果合并展示在同一个“推荐片单”中，不再按正向推荐与兜底结果拆分前端区块。

- `recommendation_score > 0` 的电影显示“推荐理由”
- 通过兜底逻辑补充进入结果的电影显示“候选依据”
- 前端不展示推荐分数

### 推荐理由定义

推荐理由用于解释：为什么这部电影符合用户偏好。

推荐理由只展示真正影响推荐分数的因素，包括：

- 命中创作者偏好
- 命中作品偏好
- 命中内容偏好
- 命中国家 / 地区偏好

不要把以下内容写进推荐理由：

- 存在几个可观看场次
- 已过滤午夜场
- 已过滤无中文字幕
- 已过滤高价场次
- 已过滤恐怖 / 惊悚片
- 已过滤短片
- 已过滤超长片

这些属于筛选过程或可用信息，不是推荐理由。

### 推荐片单中的推荐理由

当电影 `recommendation_score > 0` 时，显示：

- 推荐理由
- 可观看场次

推荐理由讲：为什么适合你。

展示示例：

```text

推荐理由：
- 命中创作者偏好：比利·怀尔德
- 命中内容偏好：名导作品
- 命中国家 / 地区偏好：美国

可观看场次：
- 6月20日 周六 18:30-20:35｜上海影城 1号厅｜IMAX / 4K
- 6月22日 周一 13:00-15:05｜大光明电影院
```

### 推荐片单中的候选依据

当电影没有命中明确偏好，但通过兜底逻辑补充进入推荐片单时，不要显示：

```text
推荐分数：0
```

应改为显示：

- 候选依据
- 可观看场次

候选依据用于解释：为什么这部电影在没有明确偏好命中时仍然进入推荐片单。

候选依据可以包含：

- 属于某个默认优先级较高的内容偏好方向
- 最早可观看场次

通过兜底逻辑补足的结果讲：为什么值得进入推荐片单。

展示示例：

```text
候选依据：
- 属于 SIFF官方精选 / 竞赛片
- 最早可观看场次：6月20日 14:00

可观看场次：
- 6月20日 周六 14:00-15:42｜上海影城 1号厅｜4K
- 6月21日 周日 19:00-20:42｜大光明电影院
```

### 场次规格展示

每部电影默认展示最早 3 个可观看场次；如果有更多场次，可以展开查看。场次按开始时间升序展示。

可观看场次中可以展示放映规格 / 设备信息。

放映规格来自 CSV 中的 `formatFlags`。

可展示的信息包括：

- IMAX
- 4K
- CINITY
- 杜比
- XR
- ONYX
- LUXE

这些信息只作为场次展示信息，不参与推荐加分。

如果没有明确放映规格，不要强行展示设备字段。

### 推荐理由数量控制

每部电影最多展示 3 条推荐理由。

推荐理由优先级：

1. 创作者 / 导演命中
2. 作品命中
3. 内容偏好命中
4. 国家 / 地区命中

如果内容偏好命中多个方向，最多展示 2 个，因为内容偏好得分最多计入 2 个方向。

### 文案原则

不要把系统筛选过程写成推荐理由。

推荐理由讲：为什么适合你。

候选依据讲：为什么值得进入推荐片单。

## 用户流程

1. 用户选择可用时间
2. 用户选择国家/地区偏好
3. 用户选择内容偏好
4. 用户选择排除项
5. 用户可选填写喜欢导演 / 创作者和印象深刻作品
6. 系统先过滤已开始场次
7. 系统执行场次级硬过滤
8. 系统按电影聚合、打分、排序
9. 如果正分结果不足，系统使用兜底逻辑补足推荐片单
10. 用户浏览推荐片单
11. 用户自行决定是否进一步了解或购票

## 前端 Demo

SIFF Picker 提供 Streamlit 前端，用于交互式生成推荐片单。

运行方式：

```bash
streamlit run streamlit_app.py
```

前端采用提交后生成模式：用户先完成筛选条件，再点击生成按钮，系统统一生成推荐片单。

前端不会在用户每次勾选条件时实时刷新推荐结果。

结果区统一显示为“推荐片单”。默认展示前 10 部，点击“查看更多”后每次继续展示 10 部；触发兜底逻辑时，最终最多展示 20 部。

## 运行模式

由于 SIFF 放映具有明确时间窗口，电影节结束后，如果始终使用真实当前北京时间，系统可能几乎没有可推荐场次。

因此前端提供两种运行模式：

1. **演示模式**
   使用固定时间 `2026-06-15 12:00 Asia/Shanghai`，用于作品集展示和产品逻辑体验。

2. **真实模式**
   使用实时当前北京时间过滤已开始场次。

CLI 默认使用真实模式。Streamlit 前端默认使用演示模式。
